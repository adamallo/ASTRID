ASTRID

ASTRID is a ILS-aware tool for species tree estimation.

How ASTRID works

ASTRID runs in two phases:

  1. a distance matrix is computed from the input gene trees
  2. a distance-based method is used to compute a species tree from the distance matrix

Installing ASTRID

To install ASTRID, clone the github repository or download it from the links at left. Then, run distmethods/install.sh. This will install the distance methods that ASTRID supports.

Currently, ASTRID supports FastME and PhyD*.

FastME

Results presented in the ASTRID paper use FASTME 1, available at http://www.ncbi.nlm.nih.gov/CBBresearch/Desper/FastME.html, but FASTME 2.0 is now available at http://www.atgc-montpellier.fr/fastme/binaries.php, and works with ASTRID as well.

PhyD*

PhyD* is available at http://www.atgc-montpellier.fr/phyd/binaries.php. It implements four algorithms: MVR, BioNJ, NJ, and UNJ. These methods are recommended for incomplete distance matrices.

Running ASTRID

ASTRID is run with the following command line arguments:

python ASTRID.py [-h] -i INPUT [-o OUTPUT] [-m METHOD] [-c CACHE]
  • METHOD is one of "auto", "bionj", "mvr", "nj", "unj", or "fastme", depending on which method you would like to use. "auto" will use FastME when the distance matrix has no missing entries, and BioNJ* if there are missing entries.
  • INPUT is a file containing a list of gene trees in newick format.
  • OUTPUT will contain the estimated species tree; the species tree will also be output to the standard output
  • CACHE is an optional argument specifying a place to save the distance matrix in PHYLIP format. This is useful if you would like to run multiple distance-based methods on the same gene tree set. If this file already exists, ASTRID will skip the matrix generation step and just run the species tree estimation step with the method specified.
